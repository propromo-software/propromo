## SCRUM Template Project V3

### Was ist neu?

Template [V2](https://github.com/orgs/propromo-software/projects/13/views/1):

- Tabs wurden verbessert
- mehr Charts
- weniger Workflows
- mehr "Custom fields"

Template [V3](https://github.com/orgs/propromo-software/projects/14/views/1):

- Tabs wurden verbessert

### Alle Versionen

<Tabs>
  <TabItem label="V1">
    **Die Version beinhaltet:** <br/>

    Views:
    * "Current Sprint (all)"
      * Board: unterteilt nach "Status"
      * Filter: `sprint:@current -status:Backlog,Updates`
      * Attribute: Title, Assignees, Status, Labels, Milestone
    * "Current Sprint (open)"
      * Board: unterteilt nach "Status"
      * Filter: `-status:Done -sprint:@previous`
      * Attribute: Title, Assignees, Status, Labels, Milestone
    * "Reviews (closed)"
      * Board: unterteilt nach "Sprint"
      * Filter: `status:Done -sprint:"dieshierwirdniemalseinsprintnamesein..."`
      * Attribute: Title, Assignees, Status, Labels, Milestone, Sprint
    * "All In One (open)"
      * Table
      * Filter: `-status:Done`
      * Attribute: Title, Assignees, Status, Labels, Milestone
    * "Swimlanes (open)"
      * Roadmap: Marker für "Milestone" und "Sprint", sortiert nach "Sprint", Datum von "Sprint", Zoom: "Month"
      * Filter: `-status:Done`
    * "Last Sprint (open)"
      * Board: unterteilt nach Status
      * Filter: `-status:Done sprint:@previous -status:Backlog -status:Updates`
      * Attribute: Title, Assignees, Status, Labels, Milestone
    * "Next Sprint (all)"
      * Board: unterteilt nach Status
      * Filter: `sprint:@next`
      * Attribute: Title, Assignees, Status, Labels, Milestone
    * "Issue Void (no sprint)"
      * Table
      * Filter: `no:sprint is:issue`
      * Attribute: Title, Assignees, Status, Labels, Milestone

    Insights:
    * "Status chart" (default) (layout: Column, x: Status, group: None, y: Count of Items),
    * "Total" (layout: Column, x: Status, group: Labels, y: Count of Items),
    * "Sprint Review 001" (layout: Bar, x: Labels, group: Assignees, y: Count of Items)

    Workflows:
    * +Pull Request (not in Sprint (Dependabot or external Contributor)) => Status: "Updates"
    * ~Issues, Pull Requests (reopened) => Status: "In Progress"
    * -Issues, Pull Requests (closed) => Status: "Done"
    * Pull Request (requires review) => Status: "Backlog"
    * Pull Request (approved) => Status: "In Progress"
    * Pull Request (merged) => Status: "Done"
    * Status: Done => close Issue

    Settings:
    * Status types: "User Stories", "ToDo", "In Progress", "Done", "Backlog", "Updates"
    * Custom fields: "Sprint" (Iteration: 2 weeks)

    *V1 ist die erste Version des Templates und somit die simpelste.*
